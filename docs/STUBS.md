# Заглушки и мок-данные

Где в проекте используются заглушки — чтобы при подключении бэкенда знать, что заменить на реальные запросы и данные.

---

## Фронт (apps/web)

| Где | Что | Файл | Замена |
|-----|-----|------|--------|
| **Профиль** | Имя, телефон, email пользователя | `pages/ProfilePage.tsx` | `mockUser` → API профиля / auth |
| **Профиль** | Числа «12 заказов», «4.9 рейтинг» | `pages/ProfilePage.tsx` | API статистики пользователя |
| **Охранники (главная)** | Список охранников | `pages/GuardsPage.tsx` | Уже есть `getNearbyGuards()` из API; при ошибке показываются `DEMO_GUARDS` |
| **Карточка охранника** | Данные одного охранника | `pages/GuardDetailPage.tsx` | ✅ Используется `GET /guards/:id`; при ошибке — мок. Телефон из API или `shared/constants.ts` (DEFAULT_PHONE). |
| **Заказы** | Список заказов | `pages/OrdersPage.tsx` | ✅ Используется `GET /orders`; при ошибке — `shared/ordersData.ts`. |
| **Детали заказа** | Один заказ по id | `pages/OrderDetailPage.tsx` | ✅ Используется `GET /orders/:id`; при ошибке — `getOrderById()`. Телефон из API или DEFAULT_PHONE. |
| **Поддержка** | Email | `pages/HelpPage.tsx` | Константа `SUPPORT_EMAIL` в `shared/constants.ts` — при необходимости заменить. |

---

## Бэкенд (backend)

| Где | Что | Файл | Замена |
|-----|-----|------|--------|
| **Список охранников** | Данные для `/guards`, `/guards/nearby`, `/guards/:id` | `handlers.rs` → `mock_guards()` | Подключение к БД; пока мок. |
| **Заказы** | Данные для `/orders`, `/orders/:id` | `handlers.rs` → `mock_orders()` | Подключение к БД; пока мок. |

---

## Экраны «скоро»

Текст «будет доступно после подключения бэкенда» или кнопка «Понятно»:

- **Безопасность** → Сессии, Пароль, 2FA (`SessionsPage`, `PasswordPage`, `TwoFAPage`) — заглушки с пояснением.
- **Премиум** (`PremiumPage`) — описание подписки, оформление «в следующем обновлении».
- **Языки** — выбор сохраняется в `localStorage` (`security_locale`), реальное переключение текстов в приложении — при появлении i18n.

---

## Настройки

- **Уведомления / Звуки** (`SettingsPage`) — переключатели работают, значение хранится в `localStorage` (`security_notifications`, `security_sounds`). При появлении бэкенда — синхронизация с профилем или API настроек.

---

Итог: да, заглушки есть — профиль, охранники (при падении API), деталка охранника, заказы, телефон и почта. Список выше можно использовать как чеклист при замене на реальный API и данные.
